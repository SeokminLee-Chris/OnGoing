## 2. 핵심 개념
=============================================================

### 2-A. representation_type (메모리 인코딩 방식)
- "perceptual" : 과거 프레임을 정적으로 sampling/dropping → static token pool
- "recurrent"  : TTT or RMT로 스트리밍 압축 → 순차적 recurrent state
- "symbolic"   : 언어 기반 subgoal (oracle/gemini/qwenvl)

### 2-B. integration_type (메모리를 모델에 넣는 방식)
- "context"    : memory token을 prefix에 단순 concatenate
- "modulation" : action token이 memory에 cross-attend → modulation signal
- "expert"     : LLM에 memory expert transformer 추가 (3-expert)

### 2-C. config 명명 규칙
  {representation}-{sampling_method}-{integration}.yaml
  예) perceptual-framesamp-modul.yaml
      recurrent-ttt-expert.yaml

## 3. 데이터 흐름
=============================================================

### 훈련 시
  Raw HDF5/Pickle (data/robomme_preprocessed_data)
    → RoboMMEDataset.__getitem__()
        └ load obs+action pair
        └ _gather_history_feat() → 과거 프레임의 image/pos/state emb 로드
    → repack_transforms (키 매핑)
    → data_transforms (delta action 계산 등 로봇별 처리)
    → Normalize(norm_stats)
    → model_transforms (resize, tokenize, pad)
    → HistAugObservation + Actions (batch)
    → train_step()
        └ 노이즈 추가 (diffusion)
        └ HistoryPi0.compute_loss()
            └ embed_prefix()  : memory + vision + language 토큰
            └ embed_suffix()  : noisy action 토큰
            └ PaliGemma forward
            └ MSE loss
        └ gradient update

### 추론 시 (serve_policy → eval client)
  관측값 dict
    → input transforms (resize, tokenize, normalize)
    → policy.add_buffer() → MemoryBuffer에 프레임 추가
    → policy.infer()
        └ _prepare_history() → 버퍼에서 memory token 준비
        └ HistoryPi0.sample_actions()
            └ 10-step reverse diffusion
    → output transforms (unnormalize)
    → actions dict

## 5. HistAugObservation 필드 구조
=============================================================

class HistAugObservation:
    # 기본 관측값
    images: dict[str, Array]           # base_0_rgb, left_wrist_0_rgb
    state: Array                        # robot joint state
    tokenized_prompt: Array

    # Perceptual memory 필드
    static_image_emb:  [b, budget, 2048]   # SigLip feature
    static_pos_emb:    [b, budget, 768]    # 3D sinusoidal pos
    static_state_emb:  [b, budget, 8]      # robot state
    static_mask:       [b, budget]

    # Recurrent memory 필드
    recur_image_emb:   [b, T, V, P, 2048]  # T=timestep, V=view, P=patch
    recur_pos_emb:     [b, T, V, P, 768]
    recur_state_emb:   [b, T, 8]
    recur_mask:        [b, T]

    # Symbolic memory 필드
    symbolic_tokenized_prompt:       [b, L]
    symbolic_tokenized_prompt_mask:  [b, L]

## 8. 중요 참고 사항
=============================================================

- 기반 모델: Pi0 (google/paligemma-3b) — openpi/ 수정 X
- 이미지 인코더 (SigLip): 기본 frozen. 훈련 시 freeze_filter 참고
- diffusion step: 기본 10 step (sample_actions)
- 데이터 전처리: 별도로 전처리 완료된 HDF5 사용
  (raw → preprocessed는 dataset_builder/ 참고)
- history_config.txt: ckpt 상위 폴더에 반드시 있어야 함
  (policy_config.py가 checkpoint_dir.parent에서 읽음)
- representation_type + integration_type 조합으로 실험 이름 결정
- norm_stats.json: runs/assets/mme_vla_suite/ 에 위치
