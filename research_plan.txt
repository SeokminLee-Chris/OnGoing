### 1. RoboMME Perceptual Memory Baseline
# Paper: "RoboMME: Benchmarking and Understanding Memory for
#          Robotic Generalist Policies" (arXiv:2603.04639)
# Base:   pi0.5 backbone, flow-matching objective
# Code:   github.com/RoboMME/robomme_policy_learning
------------------------------------------------------------

RoboMME introduces a family of perceptual memory VLA variants
built on top of pi0.5. Memory is parameterized along two axes:

[Memory Representation]

  FrameSamp:
    - 균등 간격으로 history frame 선택
    - 각 frame의 token을 max-pooling → P tokens/view/frame
    - 메모리 budget B, view 수 V → N = B/(P×V) 개 frame 사용
    - 전체 공간 context 보존

  TokenDrop:
    - 각 frame의 token을 spatial pooling → P tokens/frame
    - frame 간 평균 RGB 차이로 temporal saliency 점수 계산
    - 전체 timestep에 걸쳐 상위 B tokens 선택 (motion/변화 영역 우선)
    - Multi-modal Rotary Position Embedding (M-ROPE)으로
      시간·공간 위치 (time, height, width) 인덱스 부여

[Memory Integration]

  Context (memory-as-context):
    - memory token을 원래 입력과 concat
    - VLM expert가 jointly 처리

  Modul (memory-as-modulator):
    - action expert가 memory token에 cross-attend
    - 추출한 memory-aware 표현으로 action feature의
      LayerNorm scale/shift를 조절 (adaptive LayerNorm)

  Expert (memory-as-expert):
    - 별도 경량 memory expert가 memory token 처리
    - block-wise causal attention으로 다른 expert와 상호작용
    - VLM 동작에 간섭 최소화

[Training]
    - pi0.5 backbone 위에 end-to-end fine-tuning
    - flow-matching objective, 16개 task (770k timesteps)
    - 마지막 3 checkpoint × 3 seed = 9 run 평균

[Results]
    - Best: FrameSamp + Modul → overall 44.51%
    - 대부분의 perceptual variant가 symbolic/recurrent 최고 성능 초과
    - TokenDrop < FrameSamp: 과도한 token pruning이 global
      spatial context 손실 → distance-aware task 성능 저하

[Implementation Detail: Language Input & Chain-of-Thought]
# Confirmed from code: history_pi0.py embed_prefix() lines 468-484

    - pi0.5 원본은 two-stage: obs → text subtask 생성 →
      [obs + text subtask] → action 생성 (chain-of-thought)

    - RoboMME perceptual에서는 chain-of-thought 제거됨.
      PREFIX = [memory tokens] + [image tokens] + [tokenized_prompt]
      tokenized_prompt = task instruction만 ("clean the bedroom" 등)
      → subtask text 생성 단계 없음

    - Symbolic variant만 symbolic_tokenized_prompt 사용
      (Qwen이 예측한 subgoal text를 language input으로 사용)

    - 결과적으로 perceptual 모델은:
      task instruction + history frame만 보고
      chain-of-thought 없이 action 직접 생성해야 함
      → history를 얼마나 잘 활용하는지가 성능의 핵심

[Limitation]
    - Naive history concat 수준 (no retrieval, no consolidation)
    - 고정된 budget B로 균등 샘플링 → task-relevant frame을
      선택하는 메커니즘 없음
    - Chain-of-thought 제거로 인해 "현재 무엇을 해야 하는지"를
      perceptual history만으로 파악해야 함

### 2. MEM: Multi-Scale Embodied Memory
# Paper: "MEM: Multi-Scale Embodied Memory for Vision Language
#          Action Models" (arXiv:2603.03596)
# Base:   pi0.6 (Gemini3-4B + FAST tokens + flow-matching
#          action expert 860M params)
------------------------------------------------------------

긴 시간(최대 15분)의 task를 위해 두 가지 메모리를 결합:

[Short-horizon Memory: Video Encoder]
    - standard ViT를 space-time separable attention으로 확장
    - 4번째 레이어마다 temporal attention 추가
      (같은 patch 위치에 대해 causal masking 적용)
    - 현재 timestep의 표현만 다음으로 전달
      (과거 timestep의 patch representation은 제거)
      → single-frame VLA와 동일한 token 수 유지
    - 새로운 learnable parameter 없음
      (사전학습된 VLM 가중치로 초기화 가능)
    - 고정 sinusoidal temporal position encoding 사용

[Long-horizon Memory: Language Memory]
    - High-level policy πHL이 subtask 완료 시마다
      LLM으로 "현재까지 완료된 subtask 중 향후에도
      필요한 정보"를 압축하여 언어 메모리 mt 생성
    - 예시: 개별 색상 기억 대신 "세 개의 그릇을 cabinet에 배치"
    - 압축 이유: 반복 실패 시 naive concat은
      train-inference distribution shift 발생

[Architecture]
    - πHL (High-level): subtask instruction lt 생성 + 언어 메모리 갱신
    - πLL (Low-level): 최근 observation + lt → action 생성
    - Input: 448×448 카메라 최대 4개 + proprioceptive (linear projection)
    - Pre-train: 5 past + current = 6 frame (1초 간격)
    - Post-train: 최대 18 frame (54초)

[Training]
    - Pre-train (280k steps): 다양한 heterogeneous data
      (teleoperated demos, policy rollouts, vision-language, video-language)
    - Post-train (80k steps): flow-matching으로 continuous action

[Ablation Findings]
    - video memory 제거 시: policy가 무한 정지 (temporal grounding 상실)
    - language memory 제거 시: 먼 과거 semantic event 추적 불가
    - naive language memory (전체 subtask concat): distribution
      shift로 인해 성능 크게 저하
    - pre-training memory on diverse data: post-training-only 대비 크게 향상

[Limitation]
    - High-level / Low-level 두 policy가 필요
      (MemER처럼 완전한 planner+VLA는 아니지만 계층적)
    - Language memory 생성에 subtask annotation 필요

### 3. MemER: Scaling Up Memory via Experience Retrieval
# Paper: "MemER: Scaling Up Memory for Robot Control via
#          Experience Retrieval" (arXiv:2510.20328)
# Base:   Qwen2.5-VL-7B-Instruct (high-level) + pi0.5 (low-level)
------------------------------------------------------------

완전한 계층 구조 (Planner + VLA):

[Architecture]
    - High-level policy (1Hz): Qwen2.5-VL-7B-Instruct fine-tune
      · 입력: 최근 N frames/camera + task instruction + 선택된 keyframe들
      · 출력: language subtask + candidate keyframe 지명
    - Low-level policy (2Hz): pi0.5
      · 입력: 현재 이미지 + proprioceptive + language subtask
      · 출력: action chunk

[Keyframe Selection & Episodic Memory Consolidation]
    - 매 timestep에 high-level이 candidate keyframe index 지명
    - 누적된 index 리스트에 1D single-linkage clustering 적용
      (merge distance threshold: 5 frames)
    - 각 cluster에서 median frame index 선택 → stable episodic memory
    - context window를 벗어난 frame은 consolidated memory로 이동,
      이후 high-level policy의 입력으로 재활용

[Training]
    - High-level: task당 약 50개 demo + minimal language annotation
      vision encoder / projection layer frozen, model merging (α=0.8)
    - Low-level: pi0.5 체크포인트 + 50개 long-horizon demo
      + 10-15개 intervention example
    - Keyframe annotation: subtask 전환점 기준 semi-automatic 절차

[Limitation]
    - 두 개의 별도 모델 필요 (추론 시간, 실시간 배포 어려움)
    - task당 50개 demo + language annotation 필요
    - keyframe이 너무 많아지면 (수 시간짜리 task) 처리 불가
    - Qwen 7B inference overhead (우리가 안 가려는 방향)

### 4. MemoryVLA: Perceptual-Cognitive Memory
# Paper: "MemoryVLA: Perceptual-Cognitive Memory in VLA Models
#          for Robotic Manipulation" (arXiv:2508.19236)
# Base:   7B Prismatic VLM + diffusion transformer action expert
# Status: ICLR 2026 accepted
------------------------------------------------------------

인지과학(working memory + episodic memory) 기반 단일 모델:

[Working Memory (매 step 생성)]
    - Perceptual tokens (~256개, 256-dim):
      DINOv2 + SigLIP encoder → SE-bottleneck module로 압축
      (low-level 시각 세부 정보)
    - Cognitive tokens (단일 token):
      visual feature를 LLaMA-7B + language instruction에 통과
      (high-level semantic 요약)

[Perceptual-Cognitive Memory Bank (PCMB): 장기 저장]
    - Perceptual stream: 과거 perceptual token L개 저장
    - Cognitive stream: 과거 cognitive token L개 저장
    - 용량 초과 시 consolidation: 두 스트림 각각에서 cosine
      similarity 가장 높은 인접 entry 쌍을 평균으로 병합

[Memory Retrieval & Fusion]
    - Working memory가 PCMB에 scaled dot-product attention으로 query
    - Sinusoidal temporal positional encoding으로 "언제" 발생했는지 인식
    - Gate fusion으로 retrieved 정보와 현재 관측 통합:
        fused = gate ⊙ retrieved + (1 - gate) ⊙ current

[Action Generation]
    - Memory-conditioned diffusion transformer (action expert)
    - Cognitive token → cognition-attention layer (semantic 가이드)
    - Perceptual token → perception-attention layer (시각 세부 가이드)
    - 10 DDIM sampling steps, 16-step action sequence 생성

[Results]
    - SimplerEnv-Bridge: 71.9% (+14.6 over baseline)
    - Fractal: 72.7%
    - LIBERO-5: 96.5%
    - Real-world 12 tasks: 84.0%, long-horizon +26pp over SOTA

[Limitation]
    - LLaMA-7B로 cognitive token 생성 → 추론 overhead
    - 고정 capacity L → task 길이에 따라 조정 필요
    - VLA backbone이 pi0.5가 아닌 Prismatic 기반

## Limitations of Previous Approaches

------------------------------------------------------------
### Symbolic Memory 계열의 한계
------------------------------------------------------------

[1. Annotation Cost]

Memory를 언어 형태의 subgoal로 저장하는 Symbolic Memory 계열은 긴
작업을 계획하고 추론하는 데 강점을 보이지만, 그 대가도 분명하다.
일반적인 VLA가 (observation, task, action) 형태의 demonstration만으로
학습되는 반면, Symbolic Memory는 추가적인 subtask 또는 subgoal
annotation이 필요하다. 실제로 RoboMME는 이러한 방식이 VLM 학습을 위한
subgoal annotation을 요구한다고 명시하며, 최근 NILS와 같은 연구들 역시
인간이 부여한 instruction annotation이 데이터 스케일링의 병목이 될 수
있다고 지적한다. 결국 성능 향상과 별개로, 데이터 구축 비용 측면에서는
더 높은 진입 장벽을 가진 접근법이라 볼 수 있다.

[2. Inference Latency]

또 다른 문제는 추론 속도다. Symbolic Memory는 보통 Planner VLM이 현재
상황을 해석하고 다음 subgoal을 생성한 뒤, 이를 정책 모델이 받아 행동을
생성하는 계층적 구조를 사용한다. 이 과정은 planner inference를 추가로
필요로 하며, 실제 로봇 제어에서는 수 초 수준의 지연조차 성능 저하로
이어질 수 있다. 최근 embodied AI 연구들 역시 planner latency를 중요한
평가 요소로 다루고 있으며, 이를 줄이기 위한 다양한 방법들이 제안되고
있다. 따라서 Symbolic Memory는 강력한 reasoning 능력을 제공할 수 있지만,
annotation cost와 inference latency라는 현실적인 trade-off 역시 함께
고려해야 한다.

### Raw Demonstration Data Structure
------------------------------------------------------------

로봇 demonstration 데이터는 episode 단위로 구성:

  episode:
    task_instruction: "clean the bedroom"   ← 전 timestep 동일
    t=0:  obs_0  →  action_0
    t=1:  obs_1  →  action_1
    ...
    t=T:  obs_T  →  action_T

  obs_t = {images (multi-view), proprioceptive state}
  action_t = joint positions / end-effector commands

핵심 특성:
  - task_instruction은 episode 전체에서 고정 (subtask 변화 없음)
  - intermediate subtask label 없음
  - (obs, action) pair의 나열만 존재

→ chain-of-thought 학습을 하려면 별도 subtask annotation 필요
→ pure perceptual 방향은 raw data만으로 학습 가능 (scalable)

## Possible Ideas
=============================================================

------------------------------------------------------------
### Idea 1: Annotation-Free Event Segmentation → Keyframe Memory
# Difficulty: Medium-High / Novelty: Clear
------------------------------------------------------------

핵심 아이디어:
  - Uniform frame sampling (RoboMME FrameSamp) 대신,
    annotation 없이 event boundary를 찾아 keyframe 선택
  - 선택된 keyframe의 visual token을 memory로 사용
  - "task-relevant state transition" 지점만 저장 → efficient & semantic

Annotation-Free Event Segmentation 관련 논문 조사:
(signal 유형별 분류)

[1. Kinematic / Velocity-based]

  AWE (CoRL 2023, arXiv:2307.14326)
    - end-effector pose trajectory를 linear interpolation으로 근사
    - reconstruction error가 threshold 초과하는 지점 = keyframe
    - annotation-free, learning 불필요, 순수 kinematic 신호
    - real bimanual 로봇에서 BC horizon 최대 10x 감소, +4~28% 성능

  ORION (arXiv:2405.20321, 2024) / OKAMI (CoRL 2024, arXiv:2410.11792)
    - CoTracker로 object keypoint 추적 → velocity time-series 계산
    - ruptures 라이브러리로 kernel-based changepoint detection (PELT)
    - object contact/release/place 시점 = velocity 급변 지점
    - real robot (ORION ~69% success), humanoid (OKAMI)에 적용

[2. Visual Representation Similarity-based]

  LOTUS (ICRA 2024, arXiv:2311.02058)
    - DINOv2로 각 frame visual embedding 추출
    - bottom-up agglomerative clustering (cosine similarity)
    - 인접 segment 중 similarity 낮은 경계 = event boundary
    - real robot lifelong learning에 적용

  UBoCo (CVPR 2022, arXiv:2111.14799)
    - Temporal Self-Similarity Matrix (TSM) 구성
    - diagonal 불연속 지점 = event boundary
    - Boundary Contrastive (BoCo) loss로 encoder 학습
    - unsupervised 버전이 prior supervised SOTA 초과

[3. Prediction Error-based]

  CoSeg (TNNLS 2023, arXiv:2109.15170)
    - transformer가 주변 context로 현재 frame feature 재구성
    - reconstruction error spike = event boundary
    - 인지과학의 "예측 실패 = 사건 경계 인식"에서 영감

  Open-World SBD (ICCV 2025, arXiv:2503.10684)
    - Transformer-XL policy의 action prediction NLL error 사용
    - running average 대비 error 급증 지점 = skill boundary
    - VLA 자체의 prediction error를 재활용할 수 있다는 점에서 유망

[4. Information-theoretic]

  MaxMI (ECCV 2024, arXiv:2407.15086)
    - key state → 직전 state 간 mutual information 최대화
    - "물리적 제약이 많은 상태 = key state" (grasp, contact 등)
    - InfoNet으로 MI 추정, KSL-Net으로 soft 확률 분포 학습
    - robot manipulation sim에 직접 적용

  InfoCon (ICLR 2024, arXiv:2404.10606)
    - VQ-VAE + learnable codebook으로 manipulation concept 발견
    - generative + discriminative + actionable loss (annotation-free)
    - ManiSkill2 (Pick&Place, StackCube 등)에 적용

우리 방향에 가장 유망한 접근:
  → VLA 자체의 prediction error (SBD 방식): 별도 모듈 불필요,
    VLA backbone이 이미 학습 중이므로 재활용 가능
  → DINOv2 visual similarity (LOTUS/UBoCo 방식): vision-only,
    lightweight, pi0.5와 쉽게 통합 가능
  → Kinematic (AWE 방식): 가장 단순하지만 vision 정보 없음

Design Refinements:

  [1. Language-Conditioned Event Segmentation]
    - keyframe 선택 시 task instruction을 conditioning으로 사용 가능
    - "clean the bedroom" 같은 goal이 주어지면 어떤 state transition이
      task-relevant한지 더 잘 판단할 수 있음
    - memory 유지(event segmentation)와 action 생성은 병렬로 돌 수 있음
      → language-conditioned segmentation의 overhead가 critical path 아님

  [2. Visual Token이 아닌 압축된 표현 저장]
    - 꼭 raw visual token을 저장할 필요 없음
    - VLM EOS token, CLS token 등 이미 압축된 semantic 표현 사용 가능
    - 오래된 기억일수록 fine-grained visual detail이 덜 중요함
      (언제 무슨 일이 있었는지의 semantic만 남으면 충분)
    - 관련 선행연구:
        MemoryVLA의 cognitive token: LLaMA-7B → 단일 semantic token
        MEM의 language memory: LLM이 subtask를 자연어로 압축
    - 차이점: 우리는 annotation-free로, VLM 자체의 내부 표현을 재활용

  [3. VLM-Driven Dynamic Memory (task-adaptive, latency-free)]
    - task마다 최적 memory 전략이 다름:
        · raw visual vs VLM EOS token 중 뭐가 더 중요한지
        · 저장 주기 (매 5 step? 매 20 step?)
        · memory budget 배분
    - task 시작 전 LLM에게 task instruction을 주고 memory 전략 결정
      (task 시작 전 1회 → inference latency에 영향 없음)

    - 실행 중 VLM 역할:
        · "지금이 keyframe인가?" 판단 (event segmentation)
        · "visual token으로 저장? EOS token으로 저장?" 결정
        · action 생성과 완전히 병렬로 동작 → latency 0

    - action model은 subtask text를 절대 받지 않음:
        action input = [current obs] + [task instruction] + [compressed memory]
        subtask가 있더라도 memory management에만 사용
        → action path에 VLM latency 전혀 없음

    핵심 insight:
        기존 연구들은 subtask를 memory로 쓰면서 동시에 action conditioning에도 씀
        → VLM latency가 action 생성에 직결
        우리는 subtask/VLM을 memory 관리에만 사용
        → "memory가 충분히 좋으면 subtask로 action을 conditioning할 필요 없다"

    Concrete 2-Step Design:

      [Step 1: Pre-Episode Memory Planning (1회, amortized)]
        Input : task instruction + memory budget
        Model : LLM / VLM
        Output: "어떤 종류의 frame을 저장해야 하는가"의 기준 (criteria)
                예) "object가 grasped되는 순간"
                    "drawer가 열리는 순간"
                    "subtask 전환이 일어나는 시각적 패턴"
        비용  : episode 시작 전 1회 → inference latency에 영향 없음

      [Step 2: Per-Step Memory Filtering (매 step, parallel)]
        Input : current frame + Step 1 criteria
        Model : VLM 또는 pi0.5 자체 (이미 frame 처리 중)
        Output: yes / no → yes면 memory에 저장 (visual or EOS token)
        비용  : binary decision → lightweight 가능
                action 생성과 병렬 동작 → latency 0

      핵심 장점:
        - Step 1이 "무엇을 기억할지"의 reasoning을 1회로 amortize
        - Step 2는 단순 판단 → fast
        - annotation 불필요 (task instruction만으로 criteria 생성)
        - memory budget을 task에 맞게 동적 배분

    Core Framing:
        "We use a VLM only as an offline memory annotator, not as an online
        planner. This allows the policy to learn from task-relevant keyframe
        memories while preserving the real-time inference structure of
        standard VLA policies."

    관련 선행연구와의 차이:
        MemER:     VLM subtask → action conditioning    (latency 직접 영향)
        RoboMME symbolic: Qwen subtask → action conditioning (latency 직접 영향)
        우리:      VLM → memory management only         (action path 독립)

    Proposed Experiment (First Step):
        - MemER의 VLM (Qwen)을 keyframe selection에만 사용
        - VLA는 (obs, keyframe memory, task instruction) → action
        - subtask text는 action path에 전혀 들어가지 않음
        - RoboMME perceptual memory integration (Context/Modul/Expert) 재활용
        - 검증 hypothesis:
            "subtask conditioning 없이, 잘 선택된 keyframe memory만으로
             MemER와 동등하거나 더 나은 성능을 낼 수 있는가?"
        - latency-critical task에서 특히 이점 기대

Open Questions:
  - 어떤 신호가 VLA의 long-horizon 성능과 가장 잘 연결되는가?
  - event boundary를 온라인(inference time)으로도 탐지할 수 있는가?
  - keyframe 수를 동적으로 결정할 수 있는가?
  - EOS/CLS token vs raw visual token: 어느 시점부터 압축해도 되는가?
  - task-adaptive memory 전략이 실제로 uniform보다 얼마나 좋은가?

### Idea 2: Motion-Aware Video Encoding for Recent Frame Compression
# Difficulty: Medium / Novelty: Low (straightforward, but necessary)
------------------------------------------------------------

핵심 관찰:
  - 최근 N개 frame을 전부 VLA에 넘기는 건 비효율적
    · 인접 frame 간 mutual information이 매우 높음 (거의 동일한 scene)
    · token 수 선형 증가 → attention overhead, context window 낭비
  - 꼭 필요한 정보만 추출해서 넘겨야 함

아이디어:
  - motion-aware video encoder로 N frames를 미리 압축
  - 시간적으로 중복된 정보 제거 (mutual info 감소)
  - 변화가 있는 부분(motion, appearance change)에 집중
  - 압축된 표현만 VLA에 전달

관련 선행연구:
  - MEM: space-time separable attention ViT (temporal attention 추가,
    현재 timestep 표현만 downstream으로 전달)
  - RoboMME TokenDrop: RGB 차이로 temporal saliency 계산 후
    상위 B tokens만 선택 → 비슷한 방향이지만 단순한 heuristic 수준

Novelty 평가:
  - 아이디어 자체는 자명함 (당연한 방향)
  - MEM이 이미 유사한 접근을 함
  - 차별화하려면 "어떻게 압축하느냐"의 구체적 설계가 중요
    (motion 신호 선택, architecture, VLA 통합 방식 등)

### Idea 3: Annotation-Conditional Chain-of-Thought + Perceptual Memory
# Difficulty: High / Novelty: Clear
------------------------------------------------------------

핵심 관찰:
  - Subtask annotation 비용은 환경에 따라 다름
    · Simulation / structured tasks: annotation cheap (or already available)
    · Real-world / open-world: annotation expensive
  - RoboMME perceptual에서 pi0.5 chain-of-thought가 제거됨
    (task instruction만 language input으로 사용)

아이디어:
  - Annotation이 있을 때: chain-of-thought + perceptual memory
      [history + obs + instruction] → text subtask → action
  - Annotation이 없을 때: pure perceptual memory
      [history + obs + instruction] → action (직접)
  - 두 케이스를 하나의 모델이 커버
    → annotation 있으면 더 잘하고, 없어도 동작

Novelty:
  - pi0.5 base + perceptual memory + chain-of-thought의 조합이 없음
  - annotation 유무에 따라 gracefully degrade하는 설계
  - single model이므로 planner+VLA의 inference overhead 없음

Challenges:
  - 두 학습 regime을 하나의 모델에 통합하는 학습 방법 설계
  - chain-of-thought 유/무 간 성능 gap 측정 및 분석 필요
  - real-world 실험까지 가야 완전한 검증
