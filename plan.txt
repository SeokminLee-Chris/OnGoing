### [Phase 4] Docker 시작 시 std::system_error / Operation not permitted
- 증상: docker run 즉시 terminate called / operation not permitted
- 원인: micromamba run이 clone/unshare syscall 사용 → Docker seccomp에 막힘
- 해결: --ipc=host --security-opt seccomp=unconfined 추가, CMD를 bash로 override
- 컨테이너 진입 후 micromamba 활성화: eval "$(micromamba shell hook --shell bash)" && micromamba activate robomme

### [Phase 5] uv run 시 pynvml / mypy 다운로드 시도
- 증상: uv run scripts/xxx.py 실행 시 인터넷 없어서 패키지 다운로드 실패
- 원인: uv run은 실행 전 pyproject.toml deps 동기화 시도
- 해결: uv run → .venv/bin/python 으로 전체 교체

### [Phase 5] .venv/bin/python 실행 시 JAX not found
- 증상: python scripts/build_dataset.py 실행 시 JAX import 실패
- 원인: 시스템 python은 JAX 없음, JAX는 .venv에만 설치됨
- 해결: .venv/bin/python 직접 사용

### [Phase 5] compute_norm_stats.py 실행 시 pytest 없음
- 증상: ModuleNotFoundError: No module named 'pytest'
- 원인: Dockerfile이 uv sync --frozen --no-dev로 빌드 → pytest(dev dep) 미설치
         gemma_pytorch.py가 pytest.Cache를 타입힌트로 import
- 해결: pytest stub 생성
    SITE_PACKAGES=$(.venv/bin/python -c "import site; print(site.getsitepackages()[0])")
    echo "class Cache: pass" > $SITE_PACKAGES/pytest.py
- 주의: 컨테이너 재시작 시 날아가므로 매번 실행 필요 (Docker Quick Start 섹션 참고)

### [Phase 5] assets 폴더 컨테이너 안에서 안 보임
- 증상: cp assets/norm_stats.json 실패
- 원인: docker run 커맨드에 assets 마운트 누락
- 해결: -v /raid/seokmin/seokmin/robomme_policy_learning/assets:/app/assets 추가

### [Phase 6] 훈련 중 GPU 전체 hang (nvidia-smi도 응답 없음)
- 증상: [1]: (64,20,32)@float32 출력 후 완전 freeze, nvidia-smi hang, docker kill도 hang
- 원인: 코드가 A40(NVLink 없음) 기준으로 작성됨. A100은 NVLink 있어서 NCCL이 자동으로
         NVLink P2P 통신 시도 → deadlock. (init_train_state → jax.block_until_ready)
- 해결: NCCL_P2P_DISABLE=1 추가 → PCIe fallback으로 A40처럼 동작
- 이후 4 GPU 훈련 정상 동작 확인 ✅

### [Phase 6] tentative run 후 FileExistsError
- 증상: tentative run 완료 후 checkpoint directory already exists 에러
- 원인: train.py가 main()을 두 번 호출 (tentative → 본 훈련), 두 번째 호출 시 디렉토리 충돌
- 해결: --overwrite 플래그 추가

### [eval] Qwen subgoal predictor 동작 방식 (Phase 10)
- 매 step마다 현재 이미지 + 지금까지 예측한 subgoal history → Qwen → "다음 subgoal 하나" 실시간 예측
- 미리 전체 subgoal을 뽑아두는 것이 아님
- subgoal이 이전과 달라지면 history에 누적 → 다음 step의 context로 사용
- (예) "방 청소": step 0 → "쓰레기 집어라" 유지 → 이미지 변화 감지 → "쓰레기통에 버려라"로 전환
- 같은 subgoal이 연속으로 나와도 history에는 한 번만 추가 (마지막 항목과 다를 때만 append)
  → 100 step 동안 "쓰레기 집어라"가 나와도 history = ["쓰레기 집어라"] 유지

### [Phase 8-9] Qwen(subgoal predictor)과 PaliGemma(policy)는 완전히 별도 훈련
- Phase 8: Qwen finetune → GT subgoal 텍스트로 subgoal 예측 학습
- Phase 9: PaliGemma policy 훈련 → GT subgoal 텍스트를 language token으로 직접 입력받아 학습
  (훈련 중 Qwen은 전혀 개입하지 않음)
- Phase 10 eval 시에만 Qwen이 실시간으로 subgoal 예측 → policy에 입력
  → Qwen 예측이 틀리면 policy 입장에서 훈련 때 본 적 없는 입력이 들어오는 것 → 성능 급락 가능
  → Phase 8 Qwen finetune 품질이 Phase 10 최종 성능을 사실상 결정함
